# SocialProof

## 概要 / 개요

このリポジトリは、EC におけるソーシャルプルーフの見せ方を研究し、A/B テスト用のプロトタイプを共同で進めるための共有作業場所です。  
이 저장소는 EC 환경에서 소셜 프루프를 어떻게 제시할 것인지 연구하고, A/B 테스트용 프로토타입을 공동으로 진행하기 위한 공유 작업 공간입니다.

研究テーマは「ユーザーに近い集団によるソーシャルプルーフが、商品認知や購買判断にどのような影響を与えるか」です。  
연구 주제는 “사용자와 유사한 집단의 소셜 프루프가 상품 인식과 구매 판단에 어떤 영향을 주는가”입니다.

## 研究背景 / 연구 배경

EC では、レビュー数や人気ランキング、現在閲覧中人数などの「多数派ベースのソーシャルプルーフ」がよく使われます。  
EC에서는 리뷰 수, 인기 순위, 현재 보고 있는 사람 수와 같은 “다수 기반 소셜 프루프”가 자주 사용됩니다.

しかし研究計画では、こうした不特定多数の指標は、ユーザー自身の状況や好みに合わないと感じられると信頼されにくい可能性があるとしています。  
하지만 연구계획에서는 이러한 불특정 다수 기반 지표가 사용자 자신의 상황이나 취향과 맞지 않는다고 느껴질 경우 신뢰받기 어려울 수 있다고 봅니다.

特にファッションのように「好み」や「自分との相性」が重要な領域では、年齢、地域、属性、利用状況などが近い集団の情報のほうが、判断材料として有効かもしれません。  
특히 패션처럼 “취향”과 “나와의 적합성”이 중요한 영역에서는 연령, 지역, 속성, 이용 상황 등이 비슷한 집단의 정보가 더 유효한 판단 근거가 될 수 있습니다.

## 研究目的 / 연구 목적

この共同研究の目的は、不特定多数ベースのソーシャルプルーフよりも、ユーザーに近い集団ベースのソーシャルプルーフのほうが、信頼や説得力、購買意図にどのような違いを生むかを検証することです。  
이 공동연구의 목적은 불특정 다수 기반 소셜 프루프보다 사용자와 가까운 집단 기반 소셜 프루프가 신뢰, 설득력, 구매의도에 어떤 차이를 만드는지 검증하는 것입니다.

## 研究で扱うソーシャルプルーフ / 연구에서 다루는 소셜 프루프

- 全体人気や売れ筋のような不特定多数ベースの指標  
  전체 인기나 판매 순위와 같은 불특정 다수 기반 지표
- 地域で人気、20代女性に人気、同じ目的の人に人気、のような近い集団ベースの指標  
  지역에서 인기, 20대 여성에게 인기, 같은 목적의 사용자에게 인기와 같은 유사 집단 기반 지표
- CTA 付近に表示する短いテキスト型ソーシャルプルーフ  
  CTA 주변에 배치하는 짧은 텍스트형 소셜 프루프

## 研究計画の流れ / 연구계획의 흐름

研究計画 PDF では、次の 3 段階が示されています。  
연구계획 PDF에서는 다음의 3단계가 제시되어 있습니다.

1. 韓国と日本の EC UI 監査  
   한국과 일본의 EC UI 감사
   現在どのようなソーシャルプルーフ要素が、どこに、どの程度使われているかを整理します。  
   현재 어떤 소셜 프루프 요소가 어디에 어느 정도 사용되고 있는지 정리합니다.

2. 近い集団の知覚閾値の測定  
   가까운 집단에 대한 지각 임계값 측정
   年齢差、地域、状況の類似性など、ユーザーが「自分に近い」と感じる条件を調べます。  
   연령 차이, 지역, 상황의 유사성 등 사용자가 “나와 가깝다”고 느끼는 조건을 조사합니다.

3. 仮想 EC での A/B テスト  
   가상 EC에서의 A/B 테스트
   CTA 近くのソーシャルプルーフ表現を変えて、説得力、信頼、クリック、カート追加、購買意図への影響を比較します。  
   CTA 근처의 소셜 프루프 표현을 바꾸어 설득력, 신뢰, 클릭, 장바구니 추가, 구매의도에 미치는 영향을 비교합니다.

## 現在の段階 / 현재 단계

現在は最終プロダクトの実装段階ではなく、A/B テストを設計する段階です。  
지금은 최종 제품 구현 단계가 아니라, A/B 테스트를 설계하는 단계입니다.

このリポジトリでは特に、次の作業を進めます。  
이 저장소에서는 특히 다음 작업을 진행합니다.

- 研究計画を要約して共通理解を作る  
  연구계획을 요약해 공통 이해를 만든다
- 比較するソーシャルプルーフ表現を定義する  
  비교할 소셜 프루프 표현을 정의한다
- 実験画面の構成と文言を決める  
  실험 화면의 구성과 문구를 정한다
- 測定指標を整理する  
  측정 지표를 정리한다
- 日韓比較の観点を実験に反映する  
  한일 비교 관점을 실험에 반영한다

## フォルダ構成 / 폴더 구조

- `ab-prototype`: A/B テスト用の作業領域  
  `ab-prototype`: A/B 테스트 작업 공간
- `apps/web`: 実験後に発展させる本体アプリ領域  
  `apps/web`: 실험 이후 확장할 본체 앱 영역
- `packages/ui`: 共通 UI パーツとデザイントークン  
  `packages/ui`: 공통 UI 컴포넌트와 디자인 토큰
- `docs/product`: 研究計画の整理、要件、実験方針  
  `docs/product`: 연구계획 정리, 요구사항, 실험 방향
- `docs/design`: Figma に基づく画面整理、コンポーネント整理  
  `docs/design`: Figma 기반 화면 정리, 컴포넌트 정리
- `assets/figma`: Figma から書き出した画像や参照素材  
  `assets/figma`: Figma에서 내보낸 이미지 및 참고 자료
- `content/research`: 研究資料、元データ、要約前の素材  
  `content/research`: 연구 자료, 원본 데이터, 요약 전 자료

## 参加者向けの使い方 / 참여자를 위한 사용 방법

研究内容、目的、仮説を確認したいときは `docs/product` を見てください。  
연구 내용, 목적, 가설을 확인하고 싶을 때는 `docs/product`를 봐 주세요.

デザインや画面構成を確認したいときは `docs/design` と `assets/figma` を見てください。  
디자인이나 화면 구성을 확인하고 싶을 때는 `docs/design`과 `assets/figma`를 봐 주세요.

A/B テストの比較案や実験設計を確認したいときは `ab-prototype` を見てください。  
A/B 테스트 비교안이나 실험 설계를 확인하고 싶을 때는 `ab-prototype`를 봐 주세요.
