# Initial Structure

This repo currently separates experiment work from long-term application code.

## Short-term

- `ab-prototype`: test brief, variant definitions, measurement plan, and prototype assets
- `docs/product`: research-plan summary and experiment requirements
- `docs/design`: Figma-derived screen and component notes

## Longer-term

- `apps/web`: product implementation after experiment scope stabilizes
- `packages/ui`: reusable UI layer shared across variants and product screens
- `assets/figma`: exported design assets
- `content/research`: raw source files and research artifacts

This keeps the A/B test phase lightweight while preserving a path into a more standard app structure later.
